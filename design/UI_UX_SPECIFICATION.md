# UI/UX Specification

## Experience principles
- The primary journey starts with the video concept, source material, creative intent, and constraints.
- The interface asks what the next best creative decision is.
- Evidence stays close to every recommendation.
- Human authority is always visible.
- AI does not masquerade as certainty.
- Complex editorial state is progressively disclosed when it helps a decision. Backend
  architecture is not exposed as navigation or setup work.

## Main navigation
Story Studio and Settings. Story, review, evidence, and export views live inside a
project. Memory, Knowledge Acquisition, entities, relationships, repositories, and
workflow runs are not primary navigation concepts.

## Required project views
Current Story, Sources, Strongest Observations, Edit Recommendations, Creative
Alternatives, Prompts, Review, Lessons, and Export. Views appear only when their backing
capability is real; unavailable future outputs are not presented as successful.

## State coverage
Every screen must define loading, empty, partial, success, failure, permission denied, stale, and unavailable-evidence states where relevant.

## Visual language
Dark-first, high contrast, restrained accent use, generous spacing, professional editorial hierarchy, purposeful motion, and no glowing AI-chat aesthetic.
