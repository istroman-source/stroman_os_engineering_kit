# UI/UX Specification

## Experience principles
- The interface asks what the next best creative decision is.
- Evidence stays close to every recommendation.
- Human authority is always visible.
- AI does not masquerade as certainty.
- Complex state is shown progressively rather than hidden.

## Main navigation
Dashboard, Projects, Knowledge, Council, Playbooks, Case Studies, Analytics, Settings. MVP navigation should not imply unfinished sections are operational.

## Required project views
Overview, Media, Transcript, Story, Characters, Council, Review, Decisions, Lessons, Export.

## State coverage
Every screen must define loading, empty, partial, success, failure, permission denied, stale, and unavailable-evidence states where relevant.

## Visual language
Dark-first, high contrast, restrained accent use, generous spacing, professional editorial hierarchy, purposeful motion, and no glowing AI-chat aesthetic.
