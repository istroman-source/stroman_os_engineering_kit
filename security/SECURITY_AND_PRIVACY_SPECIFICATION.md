# Security and Privacy Specification

## Trust boundaries
Browser, application server, worker, database, object storage, AI providers, and external integrations are separate trust boundaries.

## Mandatory controls
Server authorization, workspace-scoped queries, signed storage access, secret management, validation, secure headers, rate limits, audit logs, bounded file handling, prompt-injection defenses, retention, and deletion workflows.

## AI data exposure
Workspace owners must be able to understand which provider receives which content. Logs must not expose credentials or unnecessary project material.

## Legal readiness
Terms, privacy policy, subprocessors, consent requirements, and compliance claims require qualified legal review before public launch.
