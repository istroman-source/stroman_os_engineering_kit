# Deployment and DevOps Guide

## Environments
Local, CI, staging, and production use validated configuration and separate credentials.

## Required services
Web application, background worker, PostgreSQL, object storage, queue, error monitoring, and operational logs.

## Release process
Install, lint, type-check, unit tests, integration tests, end-to-end tests, accessibility checks, AI evaluation threshold, build, migration verification, deploy, smoke test, and rollback readiness.

## Backups
Database and object-storage backup schedules, retention, encryption, restore ownership, and regular restore rehearsals must be documented.
