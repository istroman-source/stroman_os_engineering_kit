# API Specification Principles

- Typed request and response schemas.
- Workspace context and authorization on every protected operation.
- Stable identifiers and explicit version fields.
- Pagination for collections.
- Structured error codes safe for clients.
- Idempotency keys for retryable mutations and sync operations.
- Audit events for significant changes.

## Core API areas
Authentication, workspaces, projects, clients, brand profiles, assets, transcripts, evidence, analysis runs, story candidates, characters, council reviews, decisions, rubrics, final reviews, lessons, integrations, exports, search, and operations.
