# Acceptance Tests

## Project workflow
- User can create a project
- User can enter discovery information
- Stage data autosaves
- User can accept or edit AI output
- Accepted output persists

## Creative GPS
- Query "my edit feels boring" returns a diagnosis
- Response includes likely causes, first action, tree, and resources
- User can open recommended decision tree

## Decision tree
- User can move through nodes
- Back navigation works
- Completion produces an action plan
- Run is saved to project

## Rubrics
- Criteria show 1, 5, and 10 anchors
- Weighted score calculates correctly
- Justification is required
- Evaluation history is visible

## Creative Council
- At least 5 role reviews execute independently
- Synthesis lists agreements and disagreements
- Final recommendation includes confidence

## Knowledge base
- Search returns seeded content
- Related resources display
- Filters work
- Content markdown renders safely

## Final Review
- User cannot mark project complete without a review status
- Valid statuses: Ready, Needs Revision, Hold, Rebuild
- Final decision includes written reasoning

## Accessibility
- Keyboard navigation works
- Inputs have labels
- Contrast meets WCAG AA
- Decision tree is usable without drag interactions
